# 02 — React + Next.js

**Role in the plan:** Primary in Phase 1 (after consolidating basic TS).

## Junior → Mid Checklist

- [ ] Functional components + typed props
- [ ] `useState`/`useEffect` with judgment (not `useEffect` for everything)
- [ ] Custom hooks to extract reusable logic
- [ ] `useMemo`/`useCallback` — knowing WHEN they help (not using them by reflex)
- [ ] Handling lists, keys, controlled forms
- [ ] Next.js App Router: layouts, nested routes, `loading.tsx`, `error.tsx`
- [ ] Server Components vs Client Components — when to use each
- [ ] Server-side data fetching, streaming, Suspense
- [ ] Server Actions for mutations (without intermediate API routes)
- [ ] Server state vs client state (not everything is `useState`)
- [ ] Basic accessibility (HTML semantics, labels, focus)
- [ ] Avoiding unnecessary renders (understanding the why, not just the "how")
- [ ] Tailwind for styling, without fighting global CSS
- [ ] Component tests (React Testing Library)

## Sign that you're already mid on this track
You consciously decide what runs on the server and what on the client, and
your first instinct facing a performance bug is to understand the render
tree, not blindly add `useMemo`.
