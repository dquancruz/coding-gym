# 06 — Machine Learning / AI

**Role in the plan:** Primary in Phase 3. Highest projected growth and salary premium.

The 2026 trend isn't isolated "classic ML": it's **LLMs + RAG + MLOps**, on
top of a solid foundation of classic fundamentals. This track has 3 layers,
in order.

## Layer 1 — Classic fundamentals (don't skip these)
- [ ] NumPy/pandas: cleaning and transforming data without manual loops
- [ ] Correct train/test split, avoiding data leakage
- [ ] Correct metrics for the problem (accuracy isn't always the right metric)
- [ ] scikit-learn: pipelines (`Pipeline`, `ColumnTransformer`)
- [ ] Regression, classification, clustering — when to use each
- [ ] Cross-validation, overfitting vs. underfitting

## Layer 2 — Deep learning
- [ ] PyTorch: tensors, autograd, `nn.Module`, manual training loop
- [ ] Debugging exploding/vanishing gradients
- [ ] Networks for vision (basic CNN) and sequences (RNN/Transformer, intro level)

## Layer 3 — LLMs, RAG, and MLOps (where the growing demand is)
- [ ] Hugging Face Transformers: loading a model, inference, basic fine-tuning
- [ ] Structured prompt engineering (not trial and error)
- [ ] Tool calling / function calling with LLM APIs
- [ ] Embeddings + vector DB (Chroma/Pinecone/Weaviate) for RAG
- [ ] Evaluating LLM outputs (hallucination, consistency)
- [ ] LoRA/QLoRA (efficient fine-tuning) — mid-advanced level
- [ ] Serving the model: FastAPI + Docker
- [ ] Experiment tracking (MLflow) and basic CI/CD for models

## Sign that you're already mid on this track
You can take a model or LLM flow from a notebook to an endpoint served in
Docker, with a way to measure whether the output is good — not just "it
works in my test."
