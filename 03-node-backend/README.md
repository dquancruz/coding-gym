# 03 — Node Backend

**Role in the plan:** Secondary in Phase 1.

## Junior → Mid Checklist

- [ ] A basic REST endpoint with Express/Fastify
- [ ] Layered structure: routes → controller → service → repository
- [ ] Input validation (Zod) before touching business logic
- [ ] Centralized error handling (error middleware, not scattered try/catch)
- [ ] Consistent status codes and REST responses
- [ ] Auth: JWT or sessions, password hashing (bcrypt/argon2)
- [ ] Middlewares (logging, auth, rate limiting)
- [ ] Async without leaks (no swallowed promises, no race conditions)
- [ ] Environment variables and per-environment configuration
- [ ] NestJS (optional, mid-advanced level): dependency injection, modules
- [ ] Integration tests (supertest) in addition to unit tests
- [ ] Database connection with an ORM (Prisma/TypeORM) — see track 05 SQL

## Sign that you're already mid on this track
Your API doesn't crash on a weird input: you validate at the boundary,
separate responsibilities, and an error never reaches the client as a raw
stack trace.
