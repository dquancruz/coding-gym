## Junior → Mid Checklist

- [ ] Type hints on functions and variables (`mypy` with no errors)
- [ ] `dataclasses` / Pydantic models instead of loose dicts
- [ ] Idiomatic comprehensions (list/dict/set) without overusing them
- [ ] Generators and iterators for large data
- [ ] Custom context managers (`with`), not just using them
- [ ] Specific exception handling (not a generic `except Exception`)
- [ ] Modules and packages, relative vs. absolute imports
- [ ] Virtual environments and dependency management (`venv`/`poetry`)
- [ ] `pytest`: fixtures, parametrize, mocks
- [ ] FastAPI: path/query params, Pydantic, dependency injection
- [ ] Custom decorators (mid-advanced level)
- [ ] Async in Python (`asyncio`) when applicable

## Sign that you're already mid on this track

translated line by line. You use type hints as living documentation, not
decoration.
