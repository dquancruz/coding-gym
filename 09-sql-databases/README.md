# 05 — SQL & Databases

**Role in the plan:** Secondary in Phase 2. Transversal to the whole plan.

## Junior → Mid Checklist

- [ ] Comfortable with `SELECT`, `WHERE`, `ORDER BY`, `LIMIT`
- [ ] JOINs (inner, left, right, full) and understanding when to use each
- [ ] Aggregations (`GROUP BY`, `HAVING`) without confusing them with `WHERE`
- [ ] Subqueries and CTEs (`WITH`) for readable queries
- [ ] Window functions (`ROW_NUMBER`, `RANK`, `LAG/LEAD`)
- [ ] Indexes: when they help, when they don't, reading an `EXPLAIN`
- [ ] Normalization (1NF–3NF) and when to denormalize on purpose
- [ ] Transactions, isolation levels (the essentials)
- [ ] Modeling relationships (1-1, 1-N, N-N) with correct foreign keys
- [ ] ORMs (Prisma/SQLAlchemy/EF Core): when to trust them and when to drop to raw SQL
- [ ] Detecting and avoiding N+1 queries
- [ ] SQL vs. NoSQL: judgment for choosing, not dogma

## Sign that you're already mid on this track
Before writing a slow query, you know how to read the execution plan and
explain why it's slow. You design the schema thinking about the queries
you're going to run, not the other way around.
