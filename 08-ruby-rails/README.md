# 10 — Ruby on Rails

**Role in the plan:** Maintenance (1 exercise/week rotating) from day one.
Bumps to secondary in Phase 2 if your goal is product/startup.

## Junior → Mid Checklist

- [ ] Rails conventions (convention over configuration) with judgment
- [ ] ActiveRecord: associations (`has_many`, `belongs_to`), validations
- [ ] Detecting and avoiding N+1 queries (`includes`, `bullet` gem)
- [ ] Services/concerns to pull logic out of "fat" models
- [ ] Thin REST controllers (logic lives in services, not the controller)
- [ ] Background jobs (Sidekiq/ActiveJob) for heavy work
- [ ] API mode (`--api`) when the frontend is separate (React/Next)
- [ ] Tests with RSpec: models, requests, factories (FactoryBot)
- [ ] Safe migrations (reversible, no downtime in production)

## Sign that you're already mid on this track
You know when following "the Rails way" saves you time and when breaking
convention is the right call — and your models don't have 300 lines of
business logic mixed with persistence.
