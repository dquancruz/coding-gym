# 08 — Java

**Rol en el plan:** Mantenimiento (1 ejercicio/semana rotando) desde el día uno.
Sube a secundario en Fase 2 si tu objetivo es enterprise/finanzas.

## Checklist Junior → Mid

- [ ] Colecciones (`List`, `Map`, `Set`) y cuándo usar cada implementación
- [ ] Streams API (`map`, `filter`, `reduce`) en vez de loops manuales
- [ ] Genéricos y wildcards (`? extends`, `? super`)
- [ ] `Optional` en vez de `null` checks por todos lados
- [ ] Excepciones checked vs unchecked, manejo correcto
- [ ] Principios SOLID aplicados, no solo recitados
- [ ] Spring Boot: inyección de dependencias, controladores REST, JPA/Hibernate
- [ ] Tests: JUnit 5 + Mockito
- [ ] Records (Java moderno) para DTOs inmutables

## Señal de que ya eres mid en este track
Diseñas con interfaces e inyección de dependencias por defecto, no porque
"así se hace en Spring", sino porque entiendes el problema que resuelve
(testabilidad, acoplamiento).
