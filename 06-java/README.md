# 08 — Java

**Role in the plan:** Maintenance (1 exercise/week rotating) from day one.
Bumps to secondary in Phase 2 if your goal is enterprise/finance.

## Junior → Mid Checklist

- [ ] Collections (`List`, `Map`, `Set`) and when to use each implementation
- [ ] Streams API (`map`, `filter`, `reduce`) instead of manual loops
- [ ] Generics and wildcards (`? extends`, `? super`)
- [ ] `Optional` instead of `null` checks everywhere
- [ ] Checked vs. unchecked exceptions, handling them correctly
- [ ] SOLID principles applied, not just recited
- [ ] Spring Boot: dependency injection, REST controllers, JPA/Hibernate
- [ ] Tests: JUnit 5 + Mockito
- [ ] Records (modern Java) for immutable DTOs

## Sign that you're already mid on this track
You design with interfaces and dependency injection by default, not because
"that's how it's done in Spring," but because you understand the problem it
solves (testability, coupling).
