# 07 — Fundamentals

**Role in the plan:** Transversal — always active, never drops to "paused."

## Junior → Mid Checklist

- [ ] Git: branches, rebase vs. merge, resolving conflicts without panic, atomic commits
- [ ] Docker: your own Dockerfile, multi-stage builds, `docker-compose` for local environments
- [ ] Testing: the test pyramid, unit vs. integration vs. e2e, when to use each
- [ ] Basic TDD (at least for critical business logic)
- [ ] CI/CD: a pipeline in GitHub Actions (lint + test + build on every push)
- [ ] Basic systems design: caching, queues, load balancing (concepts)
- [ ] Algorithmic complexity (Big-O) — recognizing it in your own code
- [ ] Classic data structures (when to use each, not just implementing them)
- [ ] Code review: giving and receiving useful technical feedback
- [ ] Documentation: a useful README, comments that explain the "why" not the "what"

## Sign that you're already mid on this track
Your code goes through CI before it reaches anyone, your PRs are reviewable
(small, with context), and when something fails in production you know
where to start looking without panicking.
