# 07 — Fundamentos

**Rol en el plan:** Transversal — siempre activo, nunca baja a "pausado".

## Checklist Junior → Mid

- [ ] Git: ramas, rebase vs merge, resolver conflictos sin pánico, commits atómicos
- [ ] Docker: Dockerfile propio, multi-stage builds, `docker-compose` para entornos locales
- [ ] Testing: pirámide de tests, unit vs integración vs e2e, cuándo cada uno
- [ ] TDD básico (al menos para lógica de negocio crítica)
- [ ] CI/CD: pipeline en GitHub Actions (lint + test + build en cada push)
- [ ] Diseño de sistemas básico: caché, colas, balanceo de carga (conceptos)
- [ ] Complejidad algorítmica (Big-O) — reconocerla en tu propio código
- [ ] Estructuras de datos clásicas (cuándo usar cada una, no solo implementarlas)
- [ ] Code review: dar y recibir feedback técnico útil
- [ ] Documentación: README útil, comentarios que explican el "por qué" no el "qué"

## Señal de que ya eres mid en este track
Tu código pasa por CI antes de llegar a nadie, tus PRs son revisables (chicos,
con contexto), y cuando algo falla en producción sabes por dónde empezar a
buscar sin pánico.
