# 01 — TypeScript

**Role in the plan:** Primary in Phase 1. Base of the entire full-stack core.

## Junior → Mid Checklist

- [ ] Solid basic types: `interface` vs `type`, unions, intersections
- [ ] Typed functions with overloads when applicable
- [ ] Generics: generic functions and types, constraints (`extends`)
- [ ] Utility types: `Partial`, `Pick`, `Omit`, `Record`, `Readonly`
- [ ] Discriminated unions to model state (not loose boolean flags)
- [ ] Type guards and narrowing (`is`, `in`, `typeof`, `instanceof`)
- [ ] `unknown` vs `any` — never `any` by default
- [ ] Conditional types and `infer` (mid-advanced level)
- [ ] Runtime validation with Zod (types don't exist at runtime)
- [ ] `tsconfig.json` in `strict` mode, understand every flag
- [ ] Properly typed async/await, handling `Promise<T>`
- [ ] Tests with Vitest/Jest over typed pure logic

## Sign that you're already mid on this track
You model the problem domain with types (discriminated unions, generics)
BEFORE writing logic, and the compiler warns you about bugs before you run
the code — you don't use `any` as an emergency exit.
